# Contributing to HomeSphere

Thank you for considering contributing to HomeSphere!

## Getting Started

1. Fork the repository
2. Create a feature branch (git checkout -b feature/amazing-feature)
3. Make your changes
4. Run tests (backend: pytest, frontend: npm test)
5. Commit your changes
6. Push to the branch (git push origin feature/amazing-feature)
7. Open a Pull Request

## Code Style

- Backend: Use black and isort for formatting
- Frontend: Use ESLint and Prettier

## Commit Messages

Use conventional commits:
- feat: new feature
- fix: bug fix
- docs: documentation
- style: formatting
- refactor: code refactor
- test: tests
- chore: maintenance

Example: feat(finance): add budget tracking

## Pull Requests

- Use descriptive titles
- Include screenshots for UI changes
- Reference related issues
- Keep PRs focused on one feature/fix

## Reporting Issues

- Use the GitHub issue template
- Include steps to reproduce
- Add screenshots if applicable
- Specify environment (OS, browser, version)
